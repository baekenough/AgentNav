## Known Issues and Caveats

1. **URL/title mismatch**: /docs/en/api/versioning has H1 title "Versions" — the URL says "versioning" but the page title is "Versions".

2. **Legacy alias paths**: Paths using `/api/admin-api/` format (found in page-level links in earlier analysis) are not in the sitemap. Canonical admin paths use `/api/admin/`.

3. **Client-side rendering**: platform.claude.com uses React. Raw HTML fetching provides H1 and meta description but no body content. Full page content requires JavaScript execution.

4. **SDK overview gaps**: CLI, Kotlin, and Terraform SDKs have 45 API reference pages each but no /docs/en/api/sdks/{sdk} overview page (unlike Python, TypeScript, Go, Java, PHP, Ruby, C#).

5. **Meta description quality**: Simple endpoints (list, retrieve operations) have minimal meta descriptions that repeat the H1 title (e.g., "List Workspaces"). Rich meta descriptions exist only for complex operations.

6. **Pages not in sitemap**: The following paths were found in page-level links but do NOT appear in the sitemap and should be treated as legacy redirects or 404s until verified:
   - /docs/en/mcp
   - /docs/en/home
   - /docs/en/agents-and-tools/mcp (distinct from /agents-and-tools/mcp-connector)
   - /docs/en/api/files-api
   - /docs/en/api/skills/list-skills (legacy; canonical: /api/beta/skills/list)
